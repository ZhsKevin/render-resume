# Nebula RAG

> 多模态知识检索与推理系统，示例项目，全部信息均为虚构。

## 项目定位

Nebula RAG 是一套面向企业知识管理的多模态检索增强生成系统。它可以处理 PDF、Word、Markdown、网页、表格、图片和演示文稿，将非结构化资料转化为可检索、可引用、可追踪的知识资产。

系统目标是让用户用自然语言查询复杂资料，并获得带来源引用、可信度提示和上下文解释的答案。

## 核心能力

- 文档解析：支持标题层级、表格、图片说明、页码和章节结构提取。
- 混合检索：结合关键词检索、向量召回、元数据过滤和语义重排。
- 多模态理解：对图片、图表和幻灯片进行摘要，补充到统一索引中。
- 引用溯源：答案中保留来源文档、段落、页码和命中片段，便于复核。
- 质量评估：通过答案相关性、事实一致性、召回覆盖率和人工反馈持续优化。

## 技术方案

系统将知识处理流程拆为采集、解析、切分、向量化、召回、重排、压缩、生成和校验九个阶段。每个阶段都有独立指标，方便定位“搜不到”“搜到了但没用”“答案不可信”等问题。

为了降低幻觉，生成前会先做上下文压缩，生成后再做引用一致性检查。对高风险问题，系统会主动提示“资料不足”，而不是强行编答案。

## 示例成果

- 支持跨文档问答，例如“对比三份方案的成本差异并列出依据”。
- 支持表格推理，例如“找出增长最快的产品线并解释原因”。
- 支持知识库运营看板，例如召回命中率、无答案问题、热门主题和低质量文档列表。

## 可展示亮点

这个项目适合展示 RAG 工程化能力：不仅要会向量数据库，还要理解文档结构、召回质量、答案可信度和知识运营闭环。
